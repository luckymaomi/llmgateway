# LLMGateway Agent 工作规约

本文件是仓库最高工作约束。`spec.md` 是当前产品事实主干，`dev.md` 是长期工程纪律；进入正式实现后，根目录 `plan.md` 是当前任务唯一执行合同。

## 接手顺序

- 每次接手都当作一个新项目，先读 `AGENTS.md`、`spec.md`、`architecture.md`、`dev.md`、命中的项目 Skill，以及存在时的当前 `plan.md`。
- 先检查 Git 状态、当前目录、入口、数据、测试、文档和外部合同，再做判断。
- 用户输入、旧讨论、历史设计、参考项目和测试绿灯都只是证据，不自动等于当前事实。
- 先 research，再判断边界，最后行动。没有完成全局核心语义调查，不修改局部实现。
- 不回滚 owner 已有改动。commit、push、发布和部署只在 owner 明确授权时执行。

## 基础与命名

- 始终使用简体中文和 owner 交流。仓库文本统一按 UTF-8 读取；终端显示乱码时先核验编码，不把显示问题误判为文件损坏。
- 在 PowerShell 中调用 Node 包管理器时使用 `npm.cmd`、`npx.cmd`、`pnpm.cmd`，避免命中受执行策略影响的脚本包装。
- 内部变量、类型、函数、文件、模块、配置键、测试和文档标题只按当前领域职责命名，不使用 `v1`、`v2`、`v3`、`new`、`old`、`legacy`、`temp`、`final` 等演进阶段或替换顺序命名。
- 外部协议路径（例如 OpenAI-compatible `/v1`）、上游官方模型名和一次性数据库迁移序号属于真实合同，不受上一条限制；不得借这些例外给内部实现套版本壳。
- 命名调整必须逐文件理解语义并检查所有消费者，禁止无审查跨文件替换、批量正则改名和大小写不受控的迁移。
- 名称必须暴露单位和语义。时间、Token、并发、次数、字节与金额不能只叫 `value`、`limit`、`timeout` 或 `amount`；金额不用浮点数表达。
- 注释解释约束、原因和非显然取舍，不复述代码。不得留下没有 owner、验收条件或当前计划落点的 TODO、占位实现和死配置。

## 生产级阶段

- 项目可以分阶段交付，但每个阶段纳入范围的能力都必须达到生产级闭环。
- “生产级”至少包括：明确 owner、完整成功路径、可解释错误、持久状态、并发边界、中断恢复、安全约束、可观测性、测试、文档和目标环境验证。
- 阶段缩小的是功能范围，不是质量标准。主干不接受 demo、临时假实现、只在内存成立的关键事实、已知破损恢复路径或“以后再补”的双轨逻辑。
- 未进入当前阶段的能力可以不实现，但不得伪造成占位成功、虚假余额、虚假健康或已支持能力。

## 全局核心语义调查

沿完整请求链调查：

```text
客户端 -> 公共协议 -> 鉴权 -> 权限与额度 -> admission/排队
       -> 路由与账号选择 -> Provider adapter -> 上游发送
       -> 流式/非流式响应 -> usage 结算 -> 日志、指标与审计
```

管理面变更必须追踪配置写入、发布、生效、回滚和数据面读取。

改动任何事实 owner 前，列出它的输入、直接消费者、持久状态、展示出口、错误与恢复边界、测试和事实文档。公共协议、配置、schema、事件、错误类型和持久状态变化必须在同一次交付中闭环。

## 事实与 owner

- 每个事实只能有一个 owner；展示层、测试和适配器不得重算或保存第二套事实。
- Public API 拥有客户端可见协议；Canonical Model 拥有内部统一语义；Provider Adapter 拥有厂商 wire 差异。
- 身份与权限、凭据池、调度、限流、韧性策略、usage/额度账本、配置发布和可观测性分别拥有自己的状态与规则。
- 通用 OpenAI-compatible 合同由通用 adapter 维护；厂商差异只进入对应 adapter 或明确的 provider policy。
- Provider 模型、额度、RPM、错误码和能力支持属于易变外部事实，接入或修改前必须依据官方资料重新核验。
- 无法权威查询的额度必须标记为未知或估算，不能伪造“剩余额度”。

## 维护性与删除

- 删除优先于兼容。不保留旧别名、兼容转发、历史包装、双 schema 读取、临时桥接或只靠测试续命的旧路径。
- 不在错误事实之上叠加补丁。事实 owner、schema、公共合同或状态机设计被证据推翻时，修改唯一源并同步重建、再生成全部消费者，删除旧路径；禁止增加例外分支、补偿层或兼容壳维持错误设计。
- 合同正确而实现局部错误时，在根因处修复并补真实行为证据；重建范围由 owner 边界决定，不借“重建”制造无关改写。
- 当前仓库尚无正式生产 schema 和必须保留的生产数据。首次生产发布前，数据库、API、配置、事件和生成合同发生不兼容变化时，直接重建当前基线并同步全部消费者；不积累过渡 migration、不保留新旧双读双写或阶段性适配层。
- 开发/测试数据不反向绑架当前模型。需要重建时只通过独立、显式、范围可核验的命令清空 LLMGateway 自己的开发/测试资源，不把破坏性清理混入普通启动、验证或升级流程。
- 正式生产数据出现前，不兼容的数据模型变化直接重建；出现正式生产数据后，使用一次性、可验证、可回滚的前向迁移，不在运行时长期维护新旧双轨。
- 只有 owner 明确确认已有必须保留的正式生产数据后，才切换到生产迁移纪律；迁移完成后仍删除旧字段、旧 API、旧配置键和旧运行路径。Git 历史负责保存演进，不由运行时代码保存历史。
- 参考仓库只用于研究。复用代码前必须确认许可证、归属、版本和维护责任，不能把第三方源码直接混入主干。
- 单一职责按事实 owner 与变化原因判断。模块或文件出现第二个独立变化原因、替其他 owner 做业务判断、领域层反向依赖 HTTP/UI/具体存储、循环依赖、同一事实重复计算，或文件超过 300 行时，必须触发职责审查。
- 职责审查的结果可以是保持、收窄接口、移动规则或拆分；不得按行数机械切文件，也不得让 handler、adapter、store、presenter 或测试成为混合业务 owner。
- 结构化数据使用结构化接口处理；错误分类服务于恢复策略，核心控制流不得依赖散落的错误字符串猜测。
- 生成文件必须能由明确 owner 和确定性命令重建，禁止手改生成物。生成源、命令和漂移检查必须进入同一验证链。
- 新依赖先确认必要性、许可证、维护状态、发布来源和跨平台支持，再进入锁文件；不从 CDN 或运行时远端加载会参与密钥、请求或管理权限处理的代码。
- 运行配置集中、显式类型化并在启动时完整校验。无效配置直接失败，不静默回退、不维护旧 env 别名、不让同一事实散落多处。

## 恶劣路径门槛

正常退出和稳定上游都不是正确性的前提。每项能力在设计、实现和验收时必须检查：接受、执行、提交、中断、失败、恢复、重放和清理。

重点覆盖重复提交、客户端断连、流中断、超时、429/5xx、上游返回畸形数据、凭据中途停用、并发额度竞争、进程强杀、主机重启、持久存储或协调设施短暂不可用，以及用户连续执行互相冲突的管理操作。

未知副作用不能盲目重放。免费池耗尽不能自动进入付费池。重复取消、恢复和清理必须幂等。详细门槛以 `dev.md` 为准。

## 测试与验收

- 测试覆盖真实产品行为，不测试口号。长期自动测试只保护核心业务结果、公共合同、持久状态、恢复边界和高风险交互；实现重构后仍必须有产品意义，不以测试数量表现能力。
- 测试必须正向证明稳定业务结果。禁止用“某文件、路由、字符串或实现不存在”证明删除或范围收敛，也禁止用关键词、文案片段、源码文本和目录形状冒充产品行为验收。
- 时间相关测试等待可观察状态或事件，不依赖固定 sleep、机器速度或碰运气重跑。
- 视觉审美仍由人类验收，Agent 不凭截图宣布颜色、间距、品牌感或整体观感合格。Web 核心路径必须使用 Playwright 在真实浏览器中验证结构、尺寸、不溢出、导航、表单、状态和交互，并保留失败证据。
- 正式实现确定技术栈时，必须建立唯一完整验证命令。先运行定向检查，再运行完整验证和当前阶段的真实核心路径验收。
- 不用复跑掩盖不稳定测试。失败必须定位并修复测试或产品的真实竞态。

## 文件职责

- `AGENTS.md`：最高工作约束。
- `spec.md`：当前产品事实、边界与不变量。
- `dev.md`：长期工程纪律和网关验收门槛。
- `architecture.md`：已确认技术栈、模块边界、运行拓扑和管理端信息架构。
- `plan.md`：正式实现阶段当前任务的单文件执行合同；无实际实现任务时不创建。
- `README.md`：面向用户和贡献者的项目入口。
- `.agents/skills/`：可执行的开发与计划方法，不保存产品运行状态。

## Skills

- 修改实现、接口、数据、测试、事实文档、依赖、构建或运行配置前，读取 `.agents/skills/llmgateway-dev/SKILL.md`。
- 中大型开发、跨模块修改、产品边界调整、架构硬化或生产级验收，同时读取 `.agents/skills/plan/SKILL.md` 并维护根目录 `plan.md`。
- 当前仅讨论或搭建长期骨架时不创建 `plan.md`。

## 安全与交付

- 不读取、打印、提交或上传真实 API Key、令牌、个人数据和私有配置；日志、错误、指标和测试夹具默认脱敏。
- 不实现批量注册、接码、养号、绕过风控、规避平台条款或突破上游额度的能力。
- 对外监听、权限扩大、生产数据变更和付费操作必须得到 owner 明确授权。
- 交付只陈述完成事实、实际验证、未验证项和剩余风险，不把计划写成完成，也不以“后续优化”代替当前阶段闭环。
