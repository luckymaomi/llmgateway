# Security Policy

## 报告安全问题

请使用 GitHub Security Advisories 的私有漏洞报告功能联系仓库维护者，不要创建公开 Issue。

报告应包含受影响的提交或版本、最小复现步骤、影响范围和已知缓解方式。提交前删除真实 API Key、访问令牌、个人数据、完整请求内容和无关私有信息。

维护者会先确认报告，再完成复现、影响判断和修复协调；在受影响用户获得合理缓解路径前，不公开漏洞细节。

## 敏感数据边界

- LLMGateway 上游 API Key、网关访问凭据和账户状态都按敏感数据处理。
- 日志、错误、测试夹具、Issue、Pull Request 和遥测不得包含真实密钥。
- 怀疑密钥泄漏时应先在对应 Provider 撤销或轮换密钥，再提交脱敏报告。
- 请求与响应正文不进入普通日志、指标、管理审计或持久存储。
- 自定义 Provider URL 属于服务端出站边界，必须执行 SSRF、重定向、DNS 重绑定和内网访问防护。
- 安全修复不能以破坏额度账本、重复上游副作用或静默丢失运行事实为代价；未知边界必须保留可审计状态。
